# Images Directory

This directory contains images and screenshots for the Product Catalog App documentation.

## Required Images

Place the following images in this directory:

### Main Images
- `app-screenshot.png` - Main application screenshot (hero image)
- `architecture-diagram.png` - Visual architecture diagram showing microservices
- `app-running.png` - Screenshot of application running locally

### Infrastructure Images
- `docker-architecture.png` - Docker container architecture diagram
- `kubernetes-deployment.png` - Kubernetes deployment diagram
- `cicd-pipeline.png` - CI/CD pipeline flow diagram
- `pipeline-status.png` - Screenshot of pipeline runs in GitHub Actions/Azure DevOps
- `monitoring-dashboard.png` - Monitoring dashboard screenshot
- `devops-stack.png` - DevOps tools and technologies stack diagram

### Application Screenshots
- `frontend-ui.png` - Frontend user interface screenshot
- `product-list.png` - Product listing page screenshot
- `product-details.png` - Product details page with ratings screenshot

### Infrastructure Screenshots
- `docker-containers.png` - Docker containers running (docker ps output or Docker Desktop)
- `k8s-pods.png` - Kubernetes pods status (kubectl get pods output or dashboard)

## Image Guidelines

- **Format**: PNG or JPG
- **Recommended Size**: 
  - Screenshots: 1920x1080 or 1280x720
  - Diagrams: 1200x800 or higher resolution
- **File Size**: Keep under 1MB per image for better loading
- **Naming**: Use lowercase with hyphens (kebab-case)

## Creating Images

### Architecture Diagram
Use tools like:
- Draw.io / diagrams.net
- Lucidchart
- Excalidraw
- Miro

### Screenshots
- Use screenshot tools (Snipping Tool, ShareX, etc.)
- Ensure clean, professional appearance
- Remove sensitive information before sharing

### Pipeline Screenshots
- Capture from GitHub Actions or Azure DevOps
- Show successful pipeline runs
- Include relevant stages and steps
